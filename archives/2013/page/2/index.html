spring 测试开发

1 使用spring3.2，shiro作为权限管理的框架，在做junit单元测试时，遇到问题

org.apache.shiro.UnavailableSecurityManagerException: No SecurityManager accessible to the calling code, either bound to the org.apache.shiro.util.ThreadContext or as a vm static singleton.  This is an invalid application configuration.

解决办法： 写一个shiro的test类

在对controller测试时，可以很方便的绑定一个shiro的subject，测试登陆用户和普通用户

2 使用annotaion 方式来ComponentScan

    @ComponentScan(basePackages = {"com.github"}, excludeFilters = {@ComponentScan.Filter(Configuration.class)} )

在进行测试的时候会出现bean还是没有注入的问题。

解决办法：在application-xx.xml文件中配置即有效

<context:component-scan base-package="com.github"></context:component-scan>

猜想 使用java的annotation替换xml中bean的配置，应该和xml 中bean的初始化顺序有关系

git 使用

1. 如何生成key

https://help.github.com/articles/generating-ssh-keys

2.0

常用命令：

-1. 下载安装git bash http://code.google.com/p/msysgit/

  1. 设置用户信息： git config --global user.email "xxx" git config --global user.name "xxx"

  2. 检出一个项目： git clone gitxxx

  3. 导入项目到eclipse 2.1 安装EGit

  4. 创建本地branch git branch newfeature

  5. 切换到新创建的branch
    git checkout newfeature

  6. 提交所做修改 5.1 获取修改的内容

     git status

    5.2 添加到commit

     git add file1 file2

    5.3 提交修改

     git commit -m "message"

  7. 回滚到指定版本 6.1 回滚到head

       git reset --hard HEAD

    6.2 回滚到指定版本

       通过git log查询各次提交的版本
   git reset --hard 98507c0010693fa1e122daae5ac32d593258c7ce

    6.3 回滚指定文件

       git checkout HEAD file/to/restore

  8. merge到本地master git checkout master git merge newfeature
